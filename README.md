Proyecto PNFI en Informática: Sistema de Base de Datos para la Biblioteca (BdS)
Institución Educativa: Nuestra Señora de Coromoto

Responsable: Alessandro Briceño

Este proyecto es una base de datos que busca resolver los problemas en procesos manuales de inventario y perdida de este mismo.

Problemas a Resolver
1. Gestión Manual Ineficiente
El control manual de los procesos de la biblioteca, como el registro de préstamos, devoluciones e inventario, presenta varios desafíos. Este enfoque genera errores, pérdida de datos y lentitud en la atención al usuario.

2. Dificultad en la Localización de Información
La ausencia de un sistema automatizado complica la búsqueda de disponibilidad de libros y el historial de usuarios, convirtiendo este proceso en una tarea prolongada y compleja.

Objetivos del Proyecto
1. Automatizar la Gestión
Diseñar e implementar una base de datos centralizada que modernice el control de inventario y optimice el flujo de préstamos en la biblioteca.

2. Optimizar las Búsquedas
Desarrollar un módulo de consulta rápida que permita encontrar libros de manera eficiente, utilizando filtros específicos como autor, fecha de lanzamiento y grado o año escolar.

3. Mejorar la Experiencia de Usuario
Facilitar el acceso a la información tanto para el personal administrativo como para la comunidad educativa que consulta el catálogo.

Personal (Usuarios del Sistema)
El software está diseñado para satisfacer las necesidades de dos perfiles clave dentro de la institución:

1. Personal Administrativo
Responsables de la gestión y mantenimiento del sistema, tendrán control sobre el inventario, registro de nuevos ejemplares, actualización de datos y supervisión general de préstamos y devoluciones en la biblioteca.

2. Personal Docente
Usuarios que podrán interactuar con el sistema para consultar la disponibilidad del material pedagógico, planificar sus lecturas de acuerdo al grado o año escolar y gestionar los recursos académicos necesarios para sus planificaciones curriculares.

3.Estudiantes
Usuarios que podran pedir prestamos de libros y quedar registrados para llevar el inventario y sus devoluciones

Objetivo Principal
Establecer un sistema que optimice la gestión de la biblioteca, mejorando la eficiencia y la experiencia del usuario.

Usuarios Clave (Roles)
Administrador: Control total del sistema, gestión de inventario y supervisión de procesos.
Cliente (Docente y Estudiantes): Consulta de disponibilidad de materiales, planificación de recursos académicos, lista de prestamos y devoluciones. 
Funcionalidades Principales (Alcance)
Registro de usuarios.
Envío de alertas automáticas para devoluciones próximas.
Consulta rápida de libros mediante filtros específicos.
Lista de estudiantes con prestamos.
