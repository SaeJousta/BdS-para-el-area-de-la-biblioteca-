# BdS-para-el-area-de-la-biblioteca-

Proyecto PNFI en Informática
Sistema de Base de Datos para la Biblioteca (BdS)
Institución Educativa: Nuestra Señora de Coromoto

1. Problemas a Resolver
Gestión Manual Ineficiente: El principal desafío es eliminar el control manual de los procesos de la biblioteca (como el registro de préstamos, devoluciones e inventario), el cual suele generar errores, pérdida de datos y lentitud en la atención.

Dificultad en la Localización de Información: La falta de un sistema automatizado hace que buscar la disponibilidad de un libro o el historial de un usuario sea un proceso tardío y complejo.

2. Objetivos del Proyecto
Automatizar la Gestión: Diseñar e implementar una base de datos centralizada que modernice el control de inventario y el flujo de préstamos de la biblioteca.

Optimizar las Búsquedas: Desarrollar un módulo de consulta rápida que permita encontrar libros de forma eficiente mediante filtros específicos como autor, fecha de lanzamiento y el grado o año escolar correspondiente.

Mejorar la Experiencia de Usuario: Facilitar el acceso a la información tanto para el personal que administra el área como para la comunidad educativa que consulta el catálogo.

3. Personal (Usuarios del Sistema)
El software está diseñado para responder a las necesidades de dos perfiles clave dentro de la institución:

Personal Administrativo: Encargados principales de la gestión y mantenimiento del sistema. Tendrán el control del inventario, el registro de nuevos ejemplares, la actualización de datos y la supervisión global de los préstamos y devoluciones en el área de la biblioteca.

Personal Docente: Usuarios que podrán interactuar con el sistema para consultar la disponibilidad del material pedagógico, planificar sus lecturas de acuerdo al grado o año escolar, y gestionar los recursos académicos necesarios para sus planificaciones curriculares.
